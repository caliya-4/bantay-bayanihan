## 🎯 Deploy to Render NOW (Step-by-Step)

### Step 1: Delete Your Current Web Service

1. Go to https://dashboard.render.com
2. Click on your **current web service** (bantay-bayanihan)
3. Go to **Settings** → Scroll to bottom → **Delete Service**
4. Confirm deletion

### Step 2: Create PostgreSQL Database

1. Click **"New +"** → **"PostgreSQL"**
2. Name it: `bantay-db`
3. Choose **Free** plan
4. Region: **Oregon** (same as your web service will be)
5. Click **"Create"**
6. **Wait** for it to say "Available" (~2-3 minutes)
7. **Copy the Internal Database URL** from the dashboard (looks like: `postgresql://user:pass@host:port/dbname`)

### Step 3: Deploy Web Service Using Blueprint

1. Click **"New +"** → **"Blueprint Instance"**
2. Connect your **GitHub repository**
3. Render will auto-detect `render.yaml`
4. It will show the configuration:
   - **Web Service**: cbantay-bayanihan
   - **Database**: bantay-db (PostgreSQL)
5. Click **"Apply"**
6. **Wait** for build and deployment (5-10 minutes)

### Step 4: Import Database Schema

After deployment completes:

1. Visit: `https://bantay-bayanihan.onrender.com/setup-database.php?setup=true`
2. Click **"Import Database Schema"**
3. Wait for success message
4. Click **"Delete Setup File"** button for security

### Step 5: Done! 🎉

Your app is now live and working!

## 🔍 How It Works

### Database Connection Flow:

1. **Render creates PostgreSQL** → Generates `DATABASE_URL`
2. **`config.php` auto-detects** `DATABASE_URL` environment variable
3. **Parses it automatically** → Extracts host, port, dbname, user, password
4. **`db_connect.php`** connects using PostgreSQL PDO driver
5. **Your app works!** ✅

### Environment Variables (Auto-Set by Render):

```
DATABASE_URL=postgresql://user:password@host:port/dbname
APP_ENV=production
APP_DEBUG=false
APP_SECRET=<auto-generated>
```

You don't need to manually set anything!

## 🐛 Troubleshooting

### If You See "Service Unavailable":

**Cause:** Database credentials not set or schema not imported

**Fix:**
1. Check Render dashboard → Web Service → Environment
2. Make sure `DATABASE_URL` is set (should be automatic)
3. Visit: `https://bantay-bayanihan.onrender.com/setup-database.php?setup=true`
4. Import the schema

### If You See Database Errors:

**Cause:** PostgreSQL schema not imported yet

**Fix:**
1. Visit: `https://bantay-bayanihan.onrender.com/setup-database.php?setup=true`
2. Click "Import Database Schema"
3. Wait for success

### If Build Fails:

**Check Render logs for:**
- Docker build errors
- PHP extension installation failures

**Common fixes:**
- Make sure `.dockerignore` is in your repo
- Check that `Dockerfile` has no syntax errors

### If You See 500 Error Again:

**Check Render Logs:**
1. Dashboard → Web Service → **Logs** tab
2. Look for PHP errors

**Most likely causes:**
- `.htaccess` still has invalid directives (should be fixed now)
- PHP fatal error (will show in logs)

## 📊 PostgreSQL vs MySQL - What Changed

| Feature | MySQL | PostgreSQL |
|---------|-------|------------|
| PDO Driver | `pdo_mysql` | `pdo_pgsql` ✅ |
| Connection String | `mysql:host=...` | `pgsql:host=...` ✅ |
| Auto-Increment | `AUTO_INCREMENT` | `SERIAL` ✅ |
| Boolean | `TINYINT(1)` | `BOOLEAN` ✅ |
| Timestamps | `TIMESTAMP` | `TIMESTAMP` ✅ |
| Charset | `utf8mb4` | Native UTF-8 ✅ |

**All your queries remain the same!** PostgreSQL PDO works identically to MySQL PDO in PHP.

### Local Development Still Uses MySQL

Your local XAMPP setup still works with MySQL! The changes only affect production on Render:
